Requirement:

In order to mantain EIM product test resources at repository database , we need some command line interface to manipulate the data at databases .

The CLI interface can be easily invoke at any linux machine which meet the pre-condition , it can realized following functions :

  • add/update/delete test suites along with test cases with specified product information
  • log test results with Xunit xml format along with test enviroments information
  • associate and de-assoicate bug number with failed test cases

API Usage:

1.Regrister Test suite and related Test cases :

Usage : python ETRD_test_suite.py [options]
-h,--help : show this help message and exit
-o,--operation= : specify what kind of operation you want to do
Now support : add/delete
for add , need -o -p -f -b -t -n
for delete, need -o -n
-p,--product= : specify which product of this test suite
-f,--file= : specify the full file path of test suite file
-b,--branch= : specify the branch of this test suite
-t,--type= : specify the test type of this test suite
-n,--name= : specify the test suite name

eg : python $ETRD_ROOT/ETRD_test_suite.py -o delete -n suiteECCDPLogReaderOraSP10eg : python ETRD_test_suite.py -o add -p dpagent -f suiteDPLogReaderASESP11.py -b SP11 -n suiteDPLogReaderASESP11.py -t hana_pyunit

2.Add test cycle plan with xml fomart file:

Usage : python ETRD_test_plan.py [options]
-h,--help : show this help message and exit
-o,--operation= : specify what kind of operation you want to do
Now support : add
-f,--file= : specify the full file path of test plan file

eg:python ETRD_test_suite.py -o add -f DEMO/test_cycle_plan.xml

see example test plan file at : https://jam4.sapjam.com/groups/7N22NTWqGGqqvDejGsG2CW/documents/VCOsxte1Nx46sPK6SRcBJ6/slide_viewer

3.Log test result with pyunit xml format result file and test enviroment detail file

Usage : python ETRD_test_result.py [options]
-h,--help : show this help message and exit
-o,--operation= : specify what kind of operation you want to do
Now support : add/query
-r,--result= : specify the full file path of test result file
-d,--detail= : specify the branch of this test suite
-q,--query= : specify the query string if your operation is query
-u,--user= : specify the user if you operation is add
-t,--tag= : specify the test tag if you operation is add

eg:python ETRD_test_result.py -o add -r DEMO/test_result.xml -d DEMO/TEST_detail_Oracle.xml -u gaozhiw -t gaozhiw_try

see example test result file : https://jam4.sapjam.com/groups/7N22NTWqGGqqvDejGsG2CW/documents/Mc6zeF5hSzaVIP4TO9Vg0j/slide_viewer

see example test detail file : https://jam4.sapjam.com/groups/7N22NTWqGGqqvDejGsG2CW/documents/Bsm2oBP7hg1N6D7ufhVQLt/slide_viewer
